## 📋 Methodology & Assumptions

- **What it estimates (reproduction cost):** parametric cost to reproduce the present source snapshot, not historical build cost; git history is collected for context only and does not enter the headline math.
- **Three valuation lenses (P2-14):** this is the **cost approach** (reproduction/replacement cost), distinct from **value/income** and **market/comparable** — a low-cost artifact can have high value and vice-versa (IVS 2025; FASB ASC 350-40).
- **Model:** Simplified COCOMO II (A=2.94, exponent E from project class — Simple E=1.10, kept inside the model's real range) with **5 of the 17 Effort Multipliers** mapped; the other 12 default to Nominal. The displayed "EAF" is **not a calibrated COCOMO II EAF**.
- **How accurate is this, really? (P2-5):** *uncalibrated* parametric models score ≈ **MMRE 1.0 (~100% mean error)** and **PRED(25) ≈ 0** on real datasets (the "acceptable" bar is MMRE ≤ 0.25, PRED(25) ≥ 75%; calibrated COCOMO II reaches PRED(30) ≈ 52–64%). We **cannot** report this repo's MMRE/PRED (n = 1); the figures are a property of the method class and are why the band is wide and the point is order-of-magnitude.
- **Uncertainty band (Monte-Carlo, P2-6):** the headline is the **deterministic point**; the range is a seeded 10,000-iteration Monte-Carlo P10–P90 over triangular size + productivity inputs (productivity spread = the ~100% MMRE above; rate is **not** sampled — that is the Phase-4 axis). Right-skewed, so the P50 ($111K) sits above the point — a **model-uncertainty** band, not an elicited risk distribution. Replaces the old invented ×0.5–×2.0 band.
- **Estimator ensemble (P2-1/2/9/10):** structural FP/COSMIC/UCP run as **archetype-gated** cross-checks (FP/COSMIC N/A on this non-transactional compiler; UCP from the 6 CLI subcommands). SLIM is citation-only. **None move the headline**; no "convergence → confidence" claim.
- **Domain-fair complexity:** each grep factor takes `MAX(web-idiom, domain-general)` plus a depth/intensity term. This repo's Auth factor scores 5/5 from the **parser/compiler domain probe** (lexer/parser/AST/bytecode), not a web idiom — the old web-only factors would have scored it 1/5, dropping CPLX to 0.73 and the headline to ~$60K. The domain probe raises CPLX to 1.08. This is the P1 domain-fairness fix in action.
- **Measured vs parametric (P2-11/13):** N/A here (throwaway history < 10 commits); on a real repo a git session reconstruction + AI-provenance share + commit cadence are reported as a relative, unvalidated anchor, never in the headline.
- **Headline is code-only; intellectual effort is SEPARATE:** here there are no significant non-code artifacts, so IE is zero — the headline is pure COCOMO on 2 KLOC of Python.
- **Rates & overhead:** fully-loaded US consulting/agency rates (2025-2026); **no overhead multiplier** (the rate already bills it and COCOMO's `KLOC^E` already encodes size diseconomies). Team size is a schedule effect.
- **Working Hours:** 152 hours/month. **Schedule:** Tdev = 3.67 × (code-only PM)^F, F = 0.28 + 0.2×(E−0.91); Calendar Time = max(naive hours/headcount/152, Tdev). The 55/20/12/8/5 activity split is a generic industry default, not measured.
- **Calibration note:** this fixture is shipped to let readers compare the tool against a neutral, code-dominated case; **no external/market build cost is claimed for it.**
